## 4. The App (`app/index.html`)

### 4.1 Architecture
- Single-file HTML, ~1700 lines
- Vanilla JavaScript, no framework, no build step
- Firebase compat SDK v10.14.1 loaded from gstatic CDN (auth + firestore + app)
- Google Identity Services script loaded from accounts.google.com/gsi/client
- LocalStorage for fast cache, Firestore as source of truth
- Per-user Firestore doc: `users/{uid}` stores entire state object

### 4.2 Sidebar (left, sticky on desktop, drawer on mobile)
Top to bottom:
1. **Brand** — black rounded-square logo (white X drawn as filled parallelogram paths) + "XGrowth" wordmark, 18px
2. **Nav buttons** with 18px stroked SVG icons (Lucide-style):
   - Dashboard (grid icon)
   - Personal Brand Kit (user icon)
   - X Content Engine (pencil icon)
   - Campaign Builder (megaphone icon)
   - Website Copy (document icon)
   - Analytics Auditor (search icon)
   - Reports & Exports (line-chart icon)
3. **Spacer** (flex: 1)
4. **User chip** — avatar (36px), name + email (truncated), Sign out button (28px capsule)
5. Hidden tag holders (`#nicheTag`, `#audienceTag`, `#stageTag`, `#goalTag`) — invisible placeholders so `applyProfile()` JS still binds

### 4.3 Topbar (per-section)
- **Mobile** — hamburger button (`☰`) opens sidebar drawer
- **Dashboard view inner topbar** — section h2 left, range selector (7d/14d/30d, rectangular chip, 38px tall) + Profile button (pencil icon + "Profile" 12px label, 38px tall) right
- "Range" label and "vs prior X days" caption — both removed (self-explanatory)
- No global breadcrumb anymore — h2 of each section is the heading

### 4.4 Seven module views (`data-view` attributes)
- **dashboard** — KPIs, conversion funnel, follower velocity, quick actions, daily impressions bars, goal progress, top posts table, campaign ROI
- **brand** — Personal Brand Kit: niche statement, persona, X bio, calendar, etc.
- **content** — Multi-Platform Content Engine: hooks, threads, replies, captions
- **campaign** — Campaign Builder: launch storms, emails, ad copy, landing-page wireframes
- **copy** — Website Copy generator: Framer/HTML-ready sections
- **auditor** — Content Auditor: per-post score, hook diagnosis, A/B variants
- **reports** — Weekly Snapshots: markdown, copy, download .md, print/PDF, channel mix, mock email send

### 4.5 Dashboard analytics features
- **5 KPI cards**: Followers, Impressions, Engagement Rate, Profile Visits, Reply Rate. Each has value, delta % vs prior period, mini sparkline
- **Time range** 7d / 14d / 30d affecting every KPI/chart
- **Conversion funnel** — Impressions → Profile Visits → CTA Clicks → Leads
- **Follower Growth Velocity** — daily net-new bars (green/red)
- **Quick Actions** card — 4 shortcut buttons to other modules
- **Daily Impressions** — 14-day bar chart with hover tooltips
- **Goal Progress** — per-goal progress bars
- **Top Posts** table — with WoW deltas
- **Campaign ROI** table — spend / leads / CPL
- **Export**: Demo data button, Reset button, JSON export, CSV export, Markdown report download

### 4.6 Auth flow
- **Auth gate** — full-screen overlay shown until Firebase confirms user is signed in. States: loading / redirecting / signin / error. Hides via `body.authed` class. `<body>:not(.authed) .app` is `display:none`
- **Sign in entry points**: landing CTAs (popup), in-app gate "Sign in with Google" button, or returning users (auto-resume from Firebase session)
- **Popup flow** (from landing): `signInWithPopup` with `prompt: select_account`, stays in user-gesture context, no cross-domain issues
- **Redirect flow** (fallback): `signInWithRedirect` when popup blocked. Tracks `redirectResultDone` flag to avoid bouncing to landing before Firebase finishes processing the auth code
- **Welcome modal** — first-time profile setup (niche, stage, target, goal). Opens after first sign-in. Closes on outside tap + Escape. Skip button balances Save & Continue at 48px
- **Sign out** — calls `fbAuth.signOut()`, redirects to `/` (landing)
- **Unauthenticated visits to `/app/`** — `window.location.replace('/')` to send them to landing (clean back-button history)

### 4.7 Firestore cloud sync
- Source of truth: `users/{uid}` document, holds entire `state` object
- Writes: debounced 700ms after any state mutation (`save()` function)
- Reads: `loadFromCloud()` on sign-in fires once, cloud wins on conflict, then localStorage is updated and applyProfile/renderAll runs
- LocalStorage keys: `foundergrowth_v1` (state), `foundergrowth_user_v1` (user)
- Sync indicator HTML/CSS removed from sidebar (was visual noise); `setSync()` function is now a silent no-op guarded by `if(!el)return`

### 4.8 Visual design
- **Theme**: pure black `#000` background with subtle grey-on-black layers (`#070707 → #0d0d0d → #141414`)
- **Borders**: `#1f1f1f`, chips `#161616`, accents white-on-grey gradients
- **Accent color**: only green `#4ade80` for positive deltas / live indicators
- **Typography**: `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` at 16px base, 1.5 line-height
- **All inputs, selects, primary `.btn`**: 48px height
- **Textareas**: 120px min-height (long-form exception)
- **Output blocks**: system-ui 18px, 1.55 line-height (mobile drops to 16px)
- **Buttons**: white-to-grey gradient on primary, transparent border on secondary, ghost variant at 40px for inline utility actions
- **Modal**: rgba(0,0,0,.78) overlay with backdrop blur, click-outside + Escape to close
- **Mobile**: sidebar becomes slide-in drawer at ≤980px, full-width grids, KPI cards collapse 5→2→1 cols

## 7. Major Decisions Log

| Decision | Why |
|---|---|
| Single-file HTML for app + landing | Easy to ship via GitHub Pages, no build step |
| Firebase compat SDK over modular | Less disruption to existing non-module JS |
| LocalStorage + Firestore both | Fast cache + cross-device sync; cloud wins on conflict |
| Auth gate with full-screen overlay | Hide dashboard entirely until authenticated (UX + security) |
| `signInWithPopup` from landing CTAs | Avoids cross-origin storage issues that break `signInWithRedirect` in strict browsers |
| `signInWithRedirect` fallback inside `/app/` | Works when popups are blocked |
| Bounce unauthenticated `/app/` visitors to `/` | Cleaner back-button history (back from Google → landing, not stuck on `/app/`) |
| No idle timeout | Default Firebase persistence is fine; user has Sign out button; can add later if needed |
| 48px input/CTA height standard | User preference for tap-friendly forms |
| System-ui font | Native feel, no font-loading overhead |
| 4:3 mockup with bottom fade | Cinematic dashboard preview that hints at depth |
| Removed pricing section | Free during beta; will rebuild when Stripe is wired |
| Removed personal-branding angle | User wanted to target digital-product makers exclusively |

## 10. Auth + Sign-in Flow Detail

This is the trickiest part, worth documenting fully:

**Path 1: Landing CTA → popup**
```
User on /
Click "Try now for Free" / "Get Started" / etc.
→ JS handler intercepts (e.preventDefault)
→ signInWithPopup(GoogleAuthProvider) — popup window opens
→ User picks Google account in popup
→ Popup closes
→ window.location.href = '/app/'
→ /app/ loads, auth gate shows briefly
→ Firebase auth state restored (from popup result)
→ onAuthStateChanged fires with user
→ body.authed added, auth gate hides, dashboard appears
```

**Path 2: Popup blocked → redirect fallback**
```
Same as above, but signInWithPopup throws 'auth/popup-blocked'
→ Fall back to: window.location.href = '/app/?signin=1'
→ /app/ loads, auth gate "Loading..."
→ JS detects ?signin=1, strips it from URL
→ getRedirectResult().finally → redirectResultDone = true
→ user is null → state machine triggers signInWithRedirect()
→ Auth gate shows "Redirecting..."
→ Full-page navigate to Google
→ User picks account
→ Google redirects to xgrowth-351de.firebaseapp.com/__/auth/handler
→ Firebase processes, redirects to /app/
→ /app/ loads, auth gate "Loading..."
→ getRedirectResult resolves with user, onAuthStateChanged fires
→ body.authed added, app revealed
```

**Path 3: Returning visitor / refresh**
```
User opens /app/ (signed in previously)
→ Auth gate "Loading..."
→ Firebase reads persisted auth state
→ onAuthStateChanged fires with user
→ body.authed added, dashboard appears
```

**Path 4: Sign out**
```
Sign out button clicked
→ fbAuth.signOut()
→ .finally: window.location.href = '/' (landing)
```

**Path 5: Direct visit to /app/ when not signed in**
```
User types /app/ in URL (no ?signin=1)
→ Auth gate "Loading..."
→ onAuthStateChanged fires with null
→ getRedirectResult.finally → redirectResultDone = true
→ runAuthGateLogic: wantsSignin is false → window.location.replace('/')
→ User lands on landing page
```
